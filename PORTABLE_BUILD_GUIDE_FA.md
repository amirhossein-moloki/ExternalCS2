# راهنمای جامع ساخت نسخه پرتابل (Portable) برای CS2 Game Helper

این راهنما به شما کمک می‌کند تا یک نسخه کاملاً مستقل، پایدار و پرتابل از پروژه بسازید که بدون نیاز به نصب هیچ‌گونه پیش‌نیازی (مثل .NET Runtime) روی سیستم‌های مختلف اجرا شود.

## ۱. پیش‌نیازهای سیستم برنامه نویس
قبل از شروع ساخت، مطمئن شوید موارد زیر روی سیستم شما نصب است:
- **.NET 8 SDK**: برای کامپایل و انتشار پروژه.
- **Visual Studio 2022** (اختیاری اما پیشنهادی): برای مدیریت بهتر کدها.

## ۲. آماده‌سازی فایل‌های جانبی (Dependencies)
برنامه برای اجرا به سه مورد حیاتی در کنار فایل اصلی نیاز دارد:

### الف) آفست‌ها (Offsets)
برنامه بدون آفست‌های به‌روز کار نمی‌کند.
1. در پوشه اصلی پروژه، پوشه‌ای به نام `offsets` بسازید.
2. فایل‌های `offsets.json` و `client_dll.json` را از منابعی مثل [a2x/cs2-offsets](https://github.com/a2x/cs2-offsets) دریافت کرده و در این پوشه قرار دهید.

### ب) دارایی‌ها (Assets)
پوشه `assets` شامل فونت‌ها و صداهای مورد نیاز برنامه است. مطمئن شوید این پوشه در ریشه پروژه وجود دارد.

### ج) فایل تنظیمات (Config)
فایل `config.json` برای ذخیره تنظیمات شماست. یک نسخه پیش‌فرض در پروژه وجود دارد.

## ۳. دستور ساخت نسخه پرتابل استیبیل
برای ساخت نسخه‌ای که تمام پیش‌نیازها را درون خود داشته باشد، ترمینال را در پوشه پروژه باز کرده و دستور زیر را اجرا کنید:

### نسخه استاندارد (CPU-based):
این نسخه روی تمام سیستم‌ها به خوبی اجرا می‌شود.
```powershell
dotnet publish CS2GameHelper.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -o ./PortableDist
```

### نسخه شتاب‌یافته (GPU/CUDA):
اگر کارت گرافیک NVIDIA دارید و می‌خواهید از قدرت آن برای هوش مصنوعی استفاده کنید:
```powershell
dotnet publish CS2GameHelper.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true -p:UseCuda=true -o ./PortableDist_GPU
```

**تشریح پارامترهای مهم:**
- `-c Release`: کد را برای بهترین عملکرد بهینه می‌کند.
- `-r win-x64`: خروجی را مخصوص ویندوز ۶۴ بیتی می‌سازد.
- `--self-contained true`: تمام فایل‌های سیستم .NET را داخل پوشه خروجی می‌ریزد تا کاربر نیازی به نصب چیزی نداشته باشد.
- `-p:PublishSingleFile=true`: تا جای ممکن فایل‌ها را در یک فایل exe ادغام می‌کند.

## ۴. ساختار نهایی پوشه توزیع
بعد از اتمام عملیات، وارد پوشه `PortableDist` شوید. ساختار پوشه شما باید به این شکل باشد تا برنامه به درستی کار کند:

```text
PortableDist/
├── CS2GameHelper.exe (فایل اصلی برنامه)
├── config.json (تنظیمات)
├── offsets/
│   ├── offsets.json
│   └── client_dll.json
└── assets/
    ├── fonts/
    └── sounds/
```

**نکته مهم:** اگر پوشه‌های `assets` یا `offsets` به طور خودکار به این پوشه کپی نشده‌اند، حتماً آن‌ها را به صورت دستی کپی کنید.

## ۵. راهنمای اجرا برای کاربر نهایی
برای اینکه برنامه به درستی و بدون مشکل (Stable) کار کند، این موارد را رعایت کنید:

1. **دسترسی مدیریت (Run as Administrator)**: به دلیل اینکه برنامه نیاز به خواندن حافظه فرآیند بازی دارد، حتماً باید با راست‌کلیک و انتخاب "Run as Administrator" اجرا شود.
2. **اعتمادسازی آنتی‌ویروس**: اکثر آنتی‌ویروس‌ها برنامه‌های Cheat یا Memory Reader را به عنوان ویروس شناسایی می‌کنند. کل پوشه برنامه را در لیست **Exclusion** آنتی‌ویروس خود قرار دهید.
3. **ترتیب اجرا**:
   - ابتدا بازی Counter-Strike 2 را اجرا کنید.
   - صبر کنید تا به منوی اصلی بازی برسید.
   - سپس فایل `CS2GameHelper.exe` را اجرا کنید.

## ۶. عیب‌یابی (Troubleshooting)
- **برنامه بلافاصله بسته می‌شود**: آفست‌ها را چک کنید. احتمالا قدیمی هستند یا فایل‌ها در پوشه درست قرار ندارند.
- **منو باز نمی‌شود**: از کلید `Insert` روی کیبورد استفاده کنید. مطمئن شوید برنامه Overlay فعال است.
- **خطای DLL**: اگر از نسخه GPU استفاده می‌کنید، باید درایورهای CUDA روی سیستم مقصد نصب باشد. در غیر این صورت از نسخه CPU استفاده کنید.
